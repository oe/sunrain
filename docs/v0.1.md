## 🌍 路由结构（使用方案 B）

采用 SEO 友好语言结构，**默认语言为英文，首页路径为 `/`**。

| 路径                    | 页面内容      | 对应语言     |
| --------------------- | --------- | -------- |
| `/`                   | 英文首页（默认）  | English  |
| `/en/guide/anxiety`   | 焦虑指南      | English  |
| `/zh/guide/anxiety`   | 焦虑指南（翻译版） | 中文       |
| `/en/resources/music` | 音乐疗愈推荐    | English  |
| `/zh/resources/music` | 中文音乐推荐    | 中文       |
| `/es/...`             | （预留）      | Spanish  |
| `/ja/...`             | （预留）      | Japanese |

**首页 `/`：**

* 实际内容为 `/en/` 的副本，作为默认语言页
* 不使用 JS 自动跳转，仅作为英文主入口
* 使用 `<link rel="alternate" hreflang="...">` 指明其他语言版本

## 💬 多语言切换实现方式

### 页面语言切换器 `<LocaleSwitcher />`：

* 显示当前语言代码，如 "EN"、"中文"
* 点击后跳转至同页面的其他语言版本（构建时生成）

### 用户语言识别：

* 在首页中（可选）弹出提示框，例如：

  > "检测到你使用中文，是否跳转到中文版？"
* 可保存用户选择至 localStorage，避免重复提示

## 🧭 SEO 配置要点

| 设置项                                            | 内容                            |
| ---------------------------------------------- | ----------------------------- |
| `<link rel="alternate" hreflang="xx">`         | 每语言页面都要包含                     |
| `<meta name="robots" content="index, follow">` | 所有语言页面开放抓取                    |
| sitemap.xml                                    | 每语言生成各自路径                     |
| canonical 链接                                   | 指向自己路径，例如 `/zh/guide/anxiety` |
| 首页 `/`                                         | 可收录，指向英文内容，集中权重               |
